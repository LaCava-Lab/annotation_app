📚 Annotation App API Documentation

This documentation lists all available API routes for the annotation platform.

🔐 Authentication

Most routes require a Bearer Token in the Authorization header after logging in.

Authorization: Bearer <token>

👤 Auth Routes

  • POST /auth/signup - Register a new user
  • POST /auth/login - Log in and receive a token
  • POST /auth/logout - Log out

👥 User Routes

  • GET /users - List all users

📄 Paper Routes

  • GET /papers - All papers (requires auth)

📰 FullText

  • GET /fulltext?filename=PMC10546312 - Get fulltext by filename

🧠 Session State

  • GET /sessions/:id - Get session state by ID
  • POST /sessions - Create a session state

🧪 Experiments

  • GET /experiments - Get all experiments
  • POST /experiments - Add experiments in bulk

🧴 Solutions

  • GET /solutions
  • POST /solutions

🧲 Baits

  • GET /baits
  • POST /baits

🧬 Interactors

  • GET /interactors
  • POST /interactors

🧪 Chemistry

  • GET /chemistrys
  • POST /chemistrys

🧠 Made with Express + Sequelize + Postgres – Test with Postman or REST Client
